组件定义和使用

Vue 的两个核心

1. MVVM 数据驱动视图---不操作DOM
2. 组件化
  • 定义组件的方式（2种）
    1. 全局组件定义在全局，在任意组件中都可以直接使用
    2. 局部组件定义在组件内部，只能在当前组件使用
    3. 建议把通用的组件定义在全局，把非通用的业务组件定义在局部

  • 组件的组成

    • 第一个参数： 组件的名称
    • 第二个参数： 选项对象

  • 注册组件

    • 你不必把每个组件都注册到全局。 你可以通过某个 Vue 实例/组件的实例选项 components 注册仅在其作用域中可用的组件

  • 组件调用

    • 组件最后要在视图中使用，所以我们需要在视图中调用
    • 如何调用： 将组件名作为标签使用

  • 组件就是一个特殊的 Vue 实例

    • template 组件的模板
    • data
    • methods
    • computed
    • directives
    • …

  • 组件使用注意事项

    • 不需要 el ，组件的 template 就是组件管理的模板
    • 组件的 template 只能有一个根节点
    • 组件的 data 必须是函数
    • 组件是私有作用域
      • 内部无法访问外部
      • 外部也无法访问内部

  • 组件的模板

    • DOM 模板
    • 字符串模板
    • .vue 单文件组件中的 template 模板

通过 Element 感受组件的威力

  • Element 是国内饿了吗前端团队基于 Vue 开发的一个知名的第三方组件库，它能帮助我们更加快速的构建 Web 应用。
  • 官网：element.eleme.io

匿名自执行函数以及传递依赖项的问题

1. 为什么把代码写到匿名自执行函数中

  • 目的：防止变量命名空间污染
    • 防止污染别的变量
    • 防止被别的变量污染
    • 表明这是一个独立的模块
  • 原理：函数有作用域

2. 为什么把window,document传递到匿名自执行函数中

  • 告诉开发人员依赖是什么
    • 在浏览器环境执行还是在服务器环境执行，可以一目了然
  • 减少作用域查找范围（次数），提高代码执行效率

个人博客

前提环境

  1. git
  2. node
  3. github

git配置以及生成ssh秘钥

  • 查看你的配置
  • git config –list
  • 在全局配置 name 和 email
  • git config -global user.name “xxx”
  • git config -global user.email “xxx@xxx”
  • 生成秘钥
  • 在命令行执行 ssh-keygen -t rsa -C “上面配置的邮箱” 回车即可
  • 注：在此路径 –> C/user（用户）/联想/.ssh/id_rsa.pub ，打开 id_rsa.pub 文件，复制文件所有内容
  • 打开github
  • settings > SSH and GPG -> new SSH key
  • title 文本框空置
  • key 粘贴刚才复制的东西

搭建博客

  1. 创建一个远程仓库，名称固定 “你的用户名.github.io”
  • 点击SSH 复制连接
  1. 本地命令行
    • 执行 npm i -g hexo-cli hexo
    • 本地创建一个文件夹
    • cd 进入文件夹
    • 打开 git bash
      • 执行 hexo init
      • 执行 npm install
      • 执行 hexo server 或者 hexo s
    • 可以在本地开启一个服务
    • 访问 localhost:4000
    • 看到本地个人博客页面
    • 执行 hexo g
    • 执行 hexo d
    • 如果执行 hexo d 的时候报错
    • 就执行 npm i hexo-deployer-git
    • 然后重复 hexo g 和 hexo d
    • 这个时候远程仓库刷新就有文件了
    • 远程仓库 点击 setting
    • 向下翻 会有一个网址
    • 网址就是 https://你的用户名.github.io/
  2. 使用hexo
    • 在本地文件夹中打开 git bash
      • 执行 hexo new 文件名
      • 在根目录下/source/posts/文件名.md
      • 编辑文件内容
      • 编辑完成以后
        • 执行 hexo g
        • 执行 hexo d
      • 网址刷新就有了（注意缓存）
