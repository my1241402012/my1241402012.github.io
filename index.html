数据类型及转换

一、JS中的数据类型

1.基本数据类型

js中的数据类型可以分为五种：number 、string 、boolean、 underfine 、null。

  • number:数字类型 ，整型浮点型都包括。
  • string：字符串类型，右数字字母字符串以及标点符号组成，必须放在单引号或者双引号中。
  • boolean：布尔类型，只有true和false两种值。
  • underfine:未定义，一般指的是已经声明，但是没有赋值的变量，如var a;
  • null:空对象类型，var a = null，和var a=””有区别;
    • 预解析—变量提升

2.复杂数据类型

  • object：对象，对象就是把一些彼此相关的属性和方法集合在一起构成的一个数据实体。
    • 在js常见的有window, document, array等。
  • NaN：他表示的是数字类型的特殊值，可以理解为Number的一种特殊类型，只有当在js运算中发生数据类型转换时提示。
    • isNaN()方法是特有的对数据进行判断的 ，如果是数字返回false，不是数字返回true。

二、JS中的数据类型的转换

在js中数据类型转换一般分为两种，即强制类型转换和隐式类型转换(利用js弱变量类型转换)。

1.强制转换

  • 利用js提供的函数parseInt() , parseFloat() , Number(), Boolean(), String()进行数据转换。

    • 顾名思义，前两个分别是对数据进行解析转换，前者是整数，后者是浮点数。

      他们解析的原则是从前往后进行解析，尽其所能。

    • 若存在有能识别的数字就解析，如果第一位就不是数字则返回NaN.

    • Number则是对整体进行判断，是数字返回数字，否则NaN.

    • Boolean() 则是对是否存在和真假进行判断进行判断。

（1）parseInt():
  • parseInt() 方法还有基模式，可以把二进制、八进制、十六进制或其他任何进制的字符串转换成整数。
  • 基是由 parseInt() 方法的第二个参数指定的，所以要解析十六进制的值，对于含有前导的数则直接进行解析需如下调用：

​ parseInt(“AA”,16);//170

​ parseInt(“10”,2);//2

​ parseInt(“10”,8);//8

（2）ParseFloat():

​ parseFloat(“12.35元”)//12.35

​ parseFloat(“12.23.122”);//12.23

​ parseFloat(“av”);//NaN

（3）Number():

​ alert(12.3元);//false

​ Number(true);//1

​ Number(“12.3.4”);//NaN

​ Number(“ “);//0

（4）Boolean():

  • 他不会对引号里面的数字进行自动进行转换。

    Boolean([])//true

    Boolean(“0”);//true

    Boolean(null)//false

（5） String():

  • String() 则是传入的所有数据类型转换为字符串。

  • 和toString()方法的区别是

    typeof String(null)//String

    typeof String(undefined)//String

2.强制转换

  • js是弱类型的脚本语言，语法相对与java等高级编程语言来说不够严格，隐式类型转换和java中大不相同。
  • 这里的隐式类型转换指的是字符串和数值类型之间的转换。
    • 在进行字符串和数字之间进行减乘除取模运算或者进行比较运算时，他会自动把字符串转换为数字。转换数字的默认方法是调用Number().
    • 进行加法运算则是将数字看成字符串进行拼接。

匿名自执行函数以及传递依赖项的问题

1. 为什么把代码写到匿名自执行函数中

  • 目的：防止变量命名空间污染
    • 防止污染别的变量
    • 防止被别的变量污染
    • 表明这是一个独立的模块
  • 原理：函数有作用域

2. 为什么把window,document传递到匿名自执行函数中

  • 告诉开发人员依赖是什么
    • 在浏览器环境执行还是在服务器环境执行，可以一目了然
  • 减少作用域查找范围（次数），提高代码执行效率

Vue组件定义和使用

Vue 的两个核心

1. MVVM 数据驱动视图---不操作DOM
2. 组件化
  • 定义组件的方式（2种）
    1. 全局组件定义在全局，在任意组件中都可以直接使用
    2. 局部组件定义在组件内部，只能在当前组件使用
    3. 建议把通用的组件定义在全局，把非通用的业务组件定义在局部

  • 组件的组成

    • 第一个参数： 组件的名称
    • 第二个参数： 选项对象

  • 注册组件

    • 你不必把每个组件都注册到全局。 你可以通过某个 Vue 实例/组件的实例选项 components 注册仅在其作用域中可用的组件

  • 组件调用

    • 组件最后要在视图中使用，所以我们需要在视图中调用
    • 如何调用： 将组件名作为标签使用

  • 组件就是一个特殊的 Vue 实例

    • template 组件的模板
    • data
    • methods
    • computed
    • directives
    • …

  • 组件使用注意事项

    • 不需要 el ，组件的 template 就是组件管理的模板
    • 组件的 template 只能有一个根节点
    • 组件的 data 必须是函数
    • 组件是私有作用域
      • 内部无法访问外部
      • 外部也无法访问内部

  • 组件的模板

    • DOM 模板
    • 字符串模板
    • .vue 单文件组件中的 template 模板
