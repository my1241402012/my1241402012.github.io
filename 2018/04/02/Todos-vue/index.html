删除任务项

  • 点击处理函数传入参数id，根据id删除当前任务项

编辑任务项

  • 双击任务项，显示为编辑状态

    • 需要用第三方变量

  • 编辑状态时显示当前的内容

  • 保存编辑时，判断非空，将新的内容赋值给当前项，清空编辑样式

显示剩余任务数

    1. 直接在模板中写内置 javascript 表达式
    1. 把逻辑封装到方法中，然后在绑定的时候调用渲染
    1. 计算属性选择对象
      • 计算属性的本质是方法，但是只能当属性来使用 不能代用
      • 所谓的计算属性就是带有行为的属性
      • 凡是可以绑定 data 属性的地方，都可以绑定使用 计算属性
      • 计算属性从使用角度讲是有值的，从本质来讲它的值来自于自己的 get 方法的返回值
      • 计算属性会把计算结果进行缓存，使用多次只会调用一次

清除所有已完成任务项

  • for循环遍历的时候会发生跳项问题，解决方法：i--，让索引 i 倒退一次，防止漏网之鱼

切换所有任务项的完成状态

  • 通过事件源对象获取到 按钮 toggle-all 的当前选中状态

  • 遍历 todos ，让 todos 的状态 = toggle-all 的状态

  • 简单的做法：通过计算属性来处理

    • 计算属性非一般属性，本身不存储任何值，它的值来自于它自己的 get 方法的返回值
  • 计算属性的完整写法：
    属性名: {
    get : function () {}, //当访问该属性的时候，会自动调用 get 方法
    set : function () {} //当为属性赋值的时候，会调用 set 方法
    }
